# Soft-reboot-fix-for-Redmi-12-fire
Модуль который исправляет софт ребуты.

--------------------------- Новый метод ---------------------------

Заменяет файл sensors.camera.light.so на пустой файл. 
Причина, по которой происходит софт ребут когда работает этот датчик, мне пока-что не понятна. Как я думаю это связано что-то с PCI шиной(может проблема вообще с самим .so файлом). В любом случае можно покопаться в dtbo.img и изменить адреса но я вряд-ли это смогу т.к там переменные используют десятичный код и я незнаю как и на какой адрес менять. 

--------------------------- Старый метод ---------------------------

Принцип его работы :
Каждые 30/25/20 секунд перезапускается сервис android.hardware.sensors@2.0-service.multihal-mediatek тем самым перезапуская все датчики и не давая сделать софт ребут.

В случае если у вашего телефона скоро произойдет софт ребут то модуль не позволит сделать это и через несколько секунд телефон снова начнет нормально работать.

Этот модуль не исправляет симптомы софт ребута так что вы можете видеть как ваш телефон не выходит из спящего режима, не работают кнопки громкости и кнопка питания ,иногда даже может всплыть окно "Приложение system не отвечает"(если закрыть то ничего не произойдет) но это исправится через несколько секунд.

ВНИМАНИЕ : Этот модуль сделан исключительно для GSI прошивок на Redmi 12 (fire). Я не несу ответственность за ваши риски.

Числа на конце имени файла означают время через которое будет перезапускатся сервис android.hardware.sensors@2.0-service.multihal-mediatek

Рекомендуется использовать : Soft-reboot-fix25.zip
